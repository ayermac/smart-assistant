<action>
Create file `src/session/store.ts` implementing `SessionStore` interface:

1. Import `SessionFile`, `SessionMeta`, `SessionStore` from `./types.js`
2. Import `resolveDataPaths` from `../config.js`
3. Import `readFile`, `writeFile`, `rename`, `mkdir`, `readdir` from `node:fs/promises`
4. Import `join` from `node:path`
5. Create `FileSessionStore` class implementing `SessionStore` interface
6. Constructor accepts optional `sessionsDir` parameter, defaults to `resolveDataPaths().sessions`
7. Implement `generateSessionId()` helper: `new Date().toISOString().replace(/:/g, "-").slice(0, 19)`
8. Implement `create()`: return new SessionFile with generated ID, current timestamps, empty messages array
9. Implement `load(sessionId)`: read file `session-{sessionId}.json`, parse JSON, return SessionFile or null if not found
10. Implement `save(sessionId, messages)`: read existing session or create new, update messages and updatedAt, use atomic write pattern (write to `.tmp` file then rename)
11. Implement `list()`: read directory, filter files matching `session-*.json`, parse and return SessionMeta array sorted by id descending (newest first)
12. Implement `getLatest()`: call list(), return first element or null

Use async/await for all file operations. Create directory with `mkdir(path, { recursive: true })` before writes.
</action>

- Method `load("nonexistent")` returns `null`
- Method `save()` uses atomic write pattern (temp file + rename)
- Method `list()` returns sessions sorted by id descending (newest first)
- Method `getLatest()` returns null when no sessions exist
