# Plan 10-02: 文件监听与增量索引

## Objective

实现 Obsidian vault 的实时文件监听和增量索引，确保知识库与 vault 内容保持同步。

## Tasks

### Task 1: 实现文件监听器

<read_first>
- src/knowledge/vector-store.ts
- src/knowledge/types.ts
</read_first>

<action>
创建 `src/knowledge/watcher.ts`，实现：

1. `VaultWatcher` 类：
   ```typescript
   interface VaultWatcherConfig {
     vaultPath: string;
     store: VectorKnowledgeStore;
     debounceMs?: number; // 默认 1000ms
   }

   class VaultWatcher {
     constructor(config: VaultWatcherConfig);
     start(): void;   // 开始监听
     stop(): void;    // 停止监听
     getStatus(): { watching: boolean; filesTracked: number };
   }
   ```

2. 使用 `chokidar` 监听 vault 目录：
   - 监听 `**/*.md` 和 `**/*.markdown` 文件
   - 忽略 `.` 开头的目录（如 `.obsidian`, `.trash`）
   - 忽略 `.git` 目录

3. 事件处理：
   - `add` → 调用 `store.indexFile(path)` 索引新文件
   - `change` → 调用 `store.reindexFile(path)` 重新索引
   - `unlink` → 调用 `store.removeFile(path)` 移除索引

4. 防抖处理：
   - 使用 `debounceMs` 合并短时间内多次变更
   - 避免编辑器保存时的重复索引
</action>

<acceptance_criteria>
- `src/knowledge/watcher.ts` 存在并导出 `VaultWatcher`, `VaultWatcherConfig`
- `VaultWatcher` 能正确监听指定目录的 Markdown 文件变更
- 创建新 `.md` 文件触发 `add` 事件并被索引
- 修改 `.md` 文件触发 `change` 事件并重新索引
- 删除 `.md` 文件触发 `unlink` 事件并移除索引
- `npm run typecheck` 通过
</acceptance_criteria>

### Task 2: 扩展 VectorKnowledgeStore 支持增量操作

<read_first>
- src/knowledge/vector-store.ts
- src/knowledge/types.ts
</read_first>

<action>
在 `src/knowledge/vector-store.ts` 中添加方法：

1. `indexFile(filePath: string): Promise<void>` — 索引单个文件
   - 读取文件内容
   - 调用 `chunkFile` 切块
   - 生成嵌入向量
   - 存储到 LanceDB

2. `reindexFile(filePath: string): Promise<void>` — 重新索引文件
   - 先调用 `removeFile(filePath)` 清理旧数据
   - 再调用 `indexFile(filePath)` 添加新数据

3. `removeFile(filePath: string): Promise<void>` — 移除文件索引
   - 从 LanceDB 删除 `sourcePath` 匹配的所有块

4. `syncVault(vaultPath: string): Promise<{ added: number; updated: number; removed: number }>` — 全量同步
   - 扫描 vault 目录所有 Markdown 文件
   - 对比 `lastModified` 时间戳
   - 仅处理变更的文件
   - 返回同步统计
</action>

<acceptance_criteria>
- `VectorKnowledgeStore` 包含 `indexFile`, `reindexFile`, `removeFile`, `syncVault` 方法
- `removeFile` 正确清理 LanceDB 中对应文件的所有块
- `syncVault` 仅处理有变更的文件（基于 mtime 对比）
- `npm run typecheck` 通过
</acceptance_criteria>
