# Plan 05-03: Implement search_knowledge Retrieval Tool

**Goal:** Create search_knowledge tool with source metadata in results and proper empty-result handling.

**Requirements:** RAG-03, RAG-04, RAG-05

## Tasks

### Task 1: Implement createSearchKnowledgeTool

<read_first>
- src/tools/memory.ts (pattern reference for tool factory, Typebox schema, result formatting)
- src/knowledge/types.ts (KnowledgeStore, SearchOptions, KnowledgeMatch)
</read_first>

<action>
Create src/tools/knowledge.ts. Define SearchKnowledgeParameters with Typebox: query (Type.String, required), limit (Type.Optional Type.Number min 1 max 20), tags (Type.Optional Type.Array(Type.String)). Define SearchKnowledgeDetails interface with results array (id, sourcePath, headingText, snippet, relevanceScore, matchReason), total (number), fromCache (boolean). Implement createSearchKnowledgeTool(store: KnowledgeStore) returning AgentTool. In execute: call store.search(params.query, { limit, tags }). If empty results, return content text "No relevant knowledge found in the local knowledge base for [query]." with details { results: [], total: 0, fromCache: true }. If results exist, format each with snippet as first 200 chars of chunk.text, build summary string with source citations. Return formatted content and details. Wrap in try/catch: on ingestion error, return error message explaining the failure.
</action>

<acceptance_criteria>

- Successful results include id, sourcePath, headingText, snippet (max 200 chars), relevanceScore, matchReason
- Empty results return: "No relevant knowledge found in the local knowledge base for [query]."
- Tool catches ingestion errors and returns explainable error message
- Tool description explains it searches local Markdown/text knowledge
- pnpm tsc --noEmit passes
