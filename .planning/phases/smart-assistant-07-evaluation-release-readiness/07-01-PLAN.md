# Plan 07-01: Build Evaluation Fixtures and Runner

## Goal

Create the evaluation harness infrastructure including fixtures directory, test data files, and the main evaluation script (`scripts/eval.ts`) to systematically test the 10 acceptance cases.

## Tasks

### T1: Create Fixtures Directory Structure
- [ ] Create `.smart-assistant/fixtures/` directory
- [ ] Create `.smart-assistant/fixtures/memory/` subdirectory
- [ ] Create `.smart-assistant/fixtures/knowledge/` subdirectory
- [ ] Create `.smart-assistant/fixtures/sessions/` subdirectory

### T2: Create Memory Fixtures
- [ ] Create `.smart-assistant/fixtures/memory/user-profile.json` with user preferences (name, favorite language, preferences)
- [ ] Create `.smart-assistant/fixtures/memory/long-term-info.json` with persistent information to remember
- [ ] Ensure fixtures use the same schema as `src/memory/store.ts`

### T3: Create Knowledge Fixtures
- [ ] Create `.smart-assistant/fixtures/knowledge/project-docs.json` with local knowledge entries
- [ ] Create `.smart-assistant/fixtures/knowledge/empty-query-result.json` for RAG miss scenario
- [ ] Ensure fixtures use the same schema as `src/knowledge/store.ts`

### T4: Create Session Fixtures
- [ ] Create `.smart-assistant/fixtures/sessions/previous-session.json` with a prior conversation state
- [ ] Include conversation history with at least 3 turns
- [ ] Include session metadata (id, created_at, last_active)

### T5: Create Evaluation Script
- [ ] Create `scripts/eval.ts` as the main evaluation runner
- [ ] Implement test case runner with timeout handling (30s threshold for long context)
- [ ] Implement result aggregation and pass/fail counting
- [ ] Output results to `.planning/evaluation-report.md`

### T6: Implement Test Cases
- [ ] Case 1: Chat response test - verify normal question gets valid response
- [ ] Case 2: Memory storage test - verify long-term info is remembered
- [ ] Case 3: Memory recall test - verify user preferences are retrieved
- [ ] Case 4: RAG retrieval test - verify local knowledge is found
- [ ] Case 5: RAG miss test - verify "I don't know" response when knowledge not found
- [ ] Case 6: Planning decomposition test - verify task is split into steps
- [ ] Case 7: Planning status update test - verify plan state changes
- [ ] Case 8: Tool failure test - verify understandable error message (placeholder for 07-02)
- [ ] Case 9: Long context test - verify response time < 30s with long messages
- [ ] Case 10: Session restore test - verify conversation continues from saved state

### T7: Create Evaluation Report Template
- [ ] Create `.planning/evaluation-report.md` with initial template structure
- [ ] Include sections for: Summary, Individual Results, Pass Rate, Recommendations

## Verification

### Acceptance Criteria
1. Fixtures directory structure exists with all subdirectories
2. All fixture files are valid JSON matching their respective store schemas
3. `npm run eval` (or equivalent) executes the evaluation script
4. Evaluation script runs all 10 test cases and outputs results
5. Evaluation report is generated at `.planning/evaluation-report.md`
6. Each test case has clear pass/fail status in the report
7. Pass rate is calculated and displayed (target: 8/10 for release)
