# Plan 08-01: Implement VectorKnowledgeStore with LanceDB

## Objective

Create a new `VectorKnowledgeStore` class that uses LanceDB for semantic vector search, reusing the existing embedding infrastructure from Memory module.

## Background

Memory module already uses LanceDB + Doubao embedding (2048-dim vectors). This plan extends the same approach to Knowledge RAG, which currently uses keyword matching.

## Tasks

<read_first>
- src/memory/vector-store.ts
- src/memory/embedding.ts
- src/knowledge/store.ts
- src/knowledge/types.ts
</read_first>

<action>
Create `src/knowledge/vector-store.ts` with:
1. `VectorKnowledgeStore` class implementing `KnowledgeStore` interface
2. Use LanceDB with table name `knowledge` (separate from `memories`)
3. Reuse `getEmbedding` and `createDefaultEmbeddingConfig` from memory module
