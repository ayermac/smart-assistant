# Plan 08-01: Install chromadb and implement Doubao embedding API client

## Objective

Install ChromaDB npm package and create a Doubao embedding API client that can generate embeddings from text using the user-specified endpoint.

## Tasks

### Task 1: Install chromadb dependency

<read_first>
- package.json
</read_first>

<action>
Run `npm install chromadb` to add the ChromaDB client library to the project dependencies.
</action>

<acceptance_criteria>
- `package.json` contains `"chromadb"` in dependencies
- `npm install` completes successfully
- `node_modules/chromadb` directory exists
</acceptance_criteria>

### Task 2: Create embedding client module

<read_first>
- src/memory/types.ts
- src/config.ts
</read_first>

<action>
Create `src/memory/embedding.ts` with:
1. `EmbeddingConfig` interface with `baseUrl`, `model`, `apiKey` fields
2. `getEmbedding(text: string, config: EmbeddingConfig): Promise<number[]>` function
3. `createDefaultEmbeddingConfig(): EmbeddingConfig` that reads from environment variables
4. Error handling for API failures with descriptive messages
</action>

<acceptance_criteria>
- `src/memory/embedding.ts` exists and exports `getEmbedding` and `createDefaultEmbeddingConfig`
- `getEmbedding` makes POST request to `${config.baseUrl}/embeddings`
- Request body includes `model` and `input` fields
- Response is parsed and embedding array is returned
- API errors are caught and logged, function throws with descriptive error
</acceptance_criteria>

### Task 3: Add environment variable configuration

<read_first>
- .env.example
- src/config.ts
</read_first>

<action>
1. Add `EMBEDDING_BASE_URL` and `EMBEDDING_MODEL` to `.env.example` with default values
2. Update `src/config.ts` to read these environment variables with defaults:
   - `EMBEDDING_BASE_URL` default: `https://ark.cn-beijing.volces.com/api/coding/v3`
   - `EMBEDDING_MODEL` default: `doubao-embedding-vision`
</action>

<acceptance_criteria>
- `.env.example` contains `EMBEDDING_BASE_URL` and `EMBEDDING_MODEL`
- `src/config.ts` exports these values with correct defaults
- API key uses existing `ANTHROPIC_API_KEY` (no new key required)
</acceptance_criteria>

## Verification

```bash
npm run build
npm run typecheck
```

## Notes

- The Doubao API uses OpenAI-compatible format
- API key is shared with the assistant (`ANTHROPIC_API_KEY`)
- No retry logic in v1 - fail fast on API errors
