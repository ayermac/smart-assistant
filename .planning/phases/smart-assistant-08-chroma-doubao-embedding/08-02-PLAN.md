# Plan 08-02: Implement VectorMemoryStore with ChromaDB integration

## Objective

Create a VectorMemoryStore class that implements the MemoryStore interface using ChromaDB for vector storage and Doubao embedding API for semantic search.

## Tasks

### Task 1: Create VectorMemoryStore class

<read_first>
- src/memory/types.ts
- src/memory/store.ts
- src/memory/embedding.ts
</read_first>

<action>
Create `src/memory/vector-store.ts` with:
1. Import `ChromaClient` from 'chromadb'
2. Import `getEmbedding` and `EmbeddingConfig` from './embedding.js'
3. `VectorMemoryStore` class implementing `MemoryStore` interface
4. Constructor accepting `EmbeddingConfig` and optional ChromaDB path
5. `init()` method to initialize ChromaDB client and create collection
6. Private `collection` property to store ChromaDB collection reference
</action>

<acceptance_criteria>
- `src/memory/vector-store.ts` exists
- `VectorMemoryStore` implements `MemoryStore` interface from `types.ts`
- Constructor accepts embedding config
- `init()` creates ChromaDB collection named 'memories'
</acceptance_criteria>

### Task 2: Implement store() method

<read_first>
- src/memory/types.ts
- src/memory/vector-store.ts
</read_first>

<action>
Implement `store(text: string, tags?: string[]): Promise<MemoryEntry>`:
1. Generate UUID for memory ID
2. Call `getEmbedding(text, config)` to get embedding vector
3. Store in ChromaDB with `collection.add()`:
   - `ids`: [id]
   - `embeddings`: [embedding]
   - `documents`: [text]
   - `metadatas`: [{ tags, createdAt, updatedAt }]
4. Return MemoryEntry object
</action>

<acceptance_criteria>
- `store()` generates embedding via Doubao API
- Embedding is stored in ChromaDB with document and metadata
- Returns valid MemoryEntry with id, text, tags, timestamps
</acceptance_criteria>

### Task 3: Implement recall() method

<read_first>
- src/memory/types.ts
- src/memory/vector-store.ts
</read_first>

<action>
Implement `recall(query: string, options?: RecallOptions): Promise<MemoryMatch[]>`:
1. Generate embedding for query text
2. Call `collection.query()` with `queryEmbeddings`
3. Map results to `MemoryMatch[]` with:
   - `entry`: MemoryEntry from document/metadata
   - `relevanceScore`: calculated from distance (1 - distance for cosine similarity)
   - `matchReason`: 'vector similarity'
4. Support `options.limit` for result count (default 5)
5. Support `options.tags` for metadata filtering
</action>

<acceptance_criteria>
- `recall()` generates query embedding
- ChromaDB query returns semantically similar documents
- Results are mapped to MemoryMatch format
- Distance is converted to relevance score
- Limit parameter is respected
</acceptance_criteria>

### Task 4: Implement remaining MemoryStore methods

<read_first>
- src/memory/types.ts
- src/memory/vector-store.ts
</read_first>

<action>
Implement remaining interface methods:
1. `get(id: string)`: Query ChromaDB by ID, return MemoryEntry or null
2. `list()`: Get all entries from collection, return sorted by createdAt
3. `delete(id: string)`: Delete from ChromaDB by ID
</action>

<acceptance_criteria>
- `get()` retrieves single memory by ID
- `list()` returns all memories sorted by createdAt descending
- `delete()` removes memory from ChromaDB
- All methods handle errors gracefully
</acceptance_criteria>
