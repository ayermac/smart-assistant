### Task 3: Implement BM25 Keyword Retrieval

<read_first>
- src/knowledge/types.ts
</read_first>

<action>
Create `src/knowledge/bm25.ts`:

1. `BM25Match` interface:
   ```typescript
   export interface BM25Match {
     chunkId: string;
     text: string;
     score: number;
   }
   ```

2. `BM25Retriever` class:
   - `index(chunks: KnowledgeChunk[]): void` — Build inverted index
   - `search(query: string, topK?: number): BM25Match[]` — BM25 search

3. Tokenization:
   - Chinese: character-by-character + bigrams
   - English: space-separated tokens
   - No external dependencies (no jieba)

4. BM25 formula: `score = IDF × TF_norm`
   - `k1 = 1.5`, `b = 0.75`
   - IDF: `log((N - n + 0.5) / (n + 0.5) + 1)`
   - TF_norm: `((k1 + 1) × tf) / (k1 × (1 - b + b × docLen / avgDocLen) + tf)`

5. In-memory storage (sufficient for typical knowledge bases)

6. Export from `src/knowledge/index.ts`
</action>

<acceptance_criteria>
- `src/knowledge/bm25.ts` exists and exports `BM25Retriever`, `BM25Match`
- BM25 search for "LLMRouter" returns chunks containing "LLMRouter"
- BM25 search for "认证" returns chunks containing "认证"
- TypeScript compiles without errors
</acceptance_criteria>
