### Task 2: Update search() for Hybrid Retrieval

<read_first>
- src/knowledge/vector-store.ts (current)
- src/knowledge/fusion.ts
</read_first>

<action>
Refactor `VectorKnowledgeStore.search()` method:

1. Current flow:
   ```typescript
   // Before
   const vector = await this.getEmbedding(query);
   const results = await this.table.vectorSearch(vector).limit(topK);
   return results;
   ```

2. New flow:
   ```typescript
   // After
   const vector = await this.getEmbedding(query);

   // Vector search (top 20)
   const vectorResults = await this.table.vectorSearch(vector).limit(20);

   // BM25 search (top 20)
   this.ensureBM25Index();
   const bm25Results = this.bm25!.search(query, 20);

   // RRF fusion
   const fused = rrfFusion(vectorResults, bm25Results, { topN: topK });

   return fused.map(f => ({
     id: f.chunkId,
     text: f.text,
     score: f.rrfScore,
     sourcePath: // preserve source metadata
   }));
   ```

3. Preserve `options.limit` parameter support
</action>

<acceptance_criteria>
- search() uses both vector and BM25 retrieval
- Results are fused using RRF
- Public `KnowledgeStore.search()` signature unchanged
- Chunks appearing in both lists score higher
- TypeScript compiles without errors
</acceptance_criteria>
