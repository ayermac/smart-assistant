# Plan: Fix ImageVector Storage Bug

## Goal

Fix the bug where `imageVector` field is always null, preventing image content from being retrieved in search results.

## Root Cause Analysis

After code review, the issue appears to be in the image embedding flow:

1. **Image Detection** (`obsidian.ts:parseImages`): Correctly parses `![alt](path)` syntax
2. **Chunk Building** (`chunker.ts:buildChunk`): Only parses images if `vaultPath` is provided
3. **Vector Generation** (`vector-store.ts:indexFile`): Only generates `imageVector` if:
   - `this.vaultPath` is set
   - `chunk.images` exists and has items
   - `schemaHasImageVector` is true

**Potential Bug Locations:**
1. `vaultPath` not passed to `VectorKnowledgeStore` constructor
2. Image path resolution fails (relative vs absolute path issues)
3. `imageToBase64()` fails silently and returns empty string
4. Multimodal embedding API call fails but error is swallowed
