# Plan: Add LangChain Document Loaders

## Goal

Add support for PDF and DOCX document formats using LangChain's document loaders, enabling indexing and retrieval of content from these common file types.

## Current State

- `chunker.ts` only supports `.md`, `.markdown`, `.txt` extensions
- No document loaders for binary formats (PDF, DOCX)
- LanceDB schema and embedding pipeline are format-agnostic

## Tasks

- [ ] 1. Install dependencies
  ```bash
  pnpm add @langchain/community @langchain/core pdf-parse mammoth
  ```
  - `@langchain/community` - LangChain document loaders
  - `@langchain/core` - Core interfaces (Document type)
  - `pdf-parse` - PDF text extraction
  - `mammoth` - DOCX to HTML/text conversion

- [ ] 2. Create loaders directory structure
  ```
  src/knowledge/loaders/
  ├── index.ts          - Loader registry and unified interface
  ├── pdf.ts            - PDF loader implementation
  ├── docx.ts           - DOCX loader implementation
  └── types.ts          - Loader interfaces and types
  ```

- [ ] 3. Implement `types.ts` - Loader interfaces
  ```typescript
  export interface LoadedDocument {
    text: string;
    metadata: {
      sourcePath: string;
      pageCount?: number;
      wordCount?: number;
    };
  }

  export interface DocumentLoader {
    load(filePath: string): Promise<LoadedDocument>;
    supports(extension: string): boolean;
  }
  ```

- [ ] 4. Implement `pdf.ts` - PDF loader
  - Use LangChain's `PDFLoader` from `@langchain/community/document_loaders/fs/pdf`
  - Extract text content page by page
  - Preserve page numbers in metadata for citation
  - Handle encrypted/corrupted PDFs gracefully

- [ ] 5. Implement `docx.ts` - DOCX loader
  - Use `mammoth` for DOCX to text conversion
  - Extract text with basic formatting preservation
  - Handle tables and lists appropriately

- [ ] 6. Implement `index.ts` - Loader registry
  - Register all loaders
  - Provide `getLoader(extension)` function
  - Export unified `loadDocument(filePath)` function

- [ ] 7. Update `chunker.ts` to support new formats
  - Add `.pdf`, `.docx` to `SUPPORTED_EXTENSIONS`
  - Add `loadDocumentContent()` function that uses loaders
  - Modify `chunkFile()` to handle loaded documents

- [ ] 8. Update `vector-store.ts` for document loading
  - Detect file type before chunking
  - Use appropriate loader for binary formats
  - Store document metadata (page numbers, etc.)

- [ ] 9. Write tests
  - Test PDF loading with sample PDF
  - Test DOCX loading with sample DOCX
  - Test chunking produces valid KnowledgeChunks
  - Test end-to-end indexing and retrieval

## Verification

- [ ] PDF file can be indexed: `indexFile("document.pdf")` succeeds
- [ ] DOCX file can be indexed: `indexFile("document.docx")` succeeds
- [ ] Search returns results from PDF content
- [ ] Search returns results from DOCX content
- [ ] Existing Markdown/text indexing still works (backward compatibility)

## Notes

- LangChain loaders return `Document[]` with page content and metadata
- PDF parsing may lose complex formatting (tables, images) - acceptable limitation
- DOCX parsing preserves text structure but not styling
- Consider adding `.pptx` support in future (lower priority)
