# Plan: Integrate LangChain Rerank

## Goal

Add a reranking step after RRF fusion to improve retrieval relevance, especially for multi-document queries where initial ranking may not reflect true relevance.

## Current State

```
User Query
    ↓
VectorKnowledgeStore.search()
    ├── Vector Search (LanceDB) → top 20
    ├── BM25 Search (lexical) → top 20
    └── RRF Fusion → top N results
```

RRF fusion combines vector and lexical scores, but doesn't consider semantic relevance between query and result.

## Target State

```
User Query
    ↓
VectorKnowledgeStore.search()
    ├── Vector Search (LanceDB) → top 20
    ├── BM25 Search (lexical) → top 20
    └── RRF Fusion → top 20 candidates
    ↓
Rerank (cross-encoder) → top N final results
```

Reranking uses a cross-encoder model to compute query-document relevance scores, providing more accurate ranking.

## Tasks

- [ ] 1. Research LangChain reranking options
  - `ContextualCompressionRetriever` with reranker
  - `CohereRerank` - Cohere's rerank API (requires API key)
  - Custom reranker using local cross-encoder model
  - Evaluate trade-offs: API cost vs local inference

- [ ] 2. Design rerank integration approach
  - Option A: Use Cohere Rerank API (simplest, requires API key)
  - Option B: Use local cross-encoder (e.g., `cross-encoder/ms-marco-MiniLM-L-6-v2`)
  - Option C: Implement custom reranker interface for flexibility
  - **Recommendation**: Start with Option C interface, implement Option A first

- [ ] 3. Create reranker module
  ```
  src/knowledge/rerank/
  ├── index.ts          - Reranker registry
  ├── types.ts          - Reranker interface
  ├── cohere.ts         - Cohere rerank implementation
  └── noop.ts           - No-op reranker (passthrough)
  ```

- [ ] 4. Implement `types.ts` - Reranker interface
  ```typescript
  export interface RerankResult {
    chunkId: string;
    text: string;
    relevanceScore: number;  // 0-1 normalized score
  }

  export interface Reranker {
    rerank(query: string, results: FusedResult[], topN?: number): Promise<RerankResult[]>;
  }
  ```

- [ ] 5. Implement `noop.ts` - Passthrough reranker
  - Used when reranking is disabled
  - Returns results with RRF scores as relevance scores

- [ ] 6. Implement `cohere.ts` - Cohere rerank
  - Use `@langchain/cohere` or direct API call
  - Only rerank top 20 candidates for efficiency
  - Handle API errors gracefully (fallback to no-op)

- [ ] 7. Update `VectorKnowledgeStore` configuration
  - Add `reranker?: Reranker` option to config
  - Add `enableRerank?: boolean` convenience option
  - Default to no-op reranker (backward compatible)

- [ ] 8. Integrate reranking in `search()` method
  ```typescript
  // After RRF fusion
  const candidates = rrfFusion(vectorMatches, bm25Results, { topN: 20 });

  // Rerank if enabled
  const reranked = await this.reranker.rerank(query, candidates, limit);

  // Return final results
  ```

- [ ] 9. Add configuration options
  - Environment variable: `RERANK_ENABLED=true/false`
  - Environment variable: `RERANK_PROVIDER=cohere|noop`
  - Environment variable: `COHERE_API_KEY` for Cohere

- [ ] 10. Write tests
  - Test no-op reranker returns unchanged results

- Cross-encoder models are more accurate but slower than bi-encoders
