# 12-01 Plan: Project Consistency Repair

## Goal

Repair consistency issues found during the project analysis without changing the assistant's intended user-facing behavior.

## Scope

- Align documented provider/model defaults with runtime defaults.
- Make `--data-dir` affect every local store created by the CLI.
- Make vector knowledge ingestion idempotent and consistent across full and incremental indexing.
- Keep Vitest discovery focused on source tests instead of compiled `dist` tests.
- Remove test-only tools from normal assistant runtime.

## Implementation Plan

1. Centralize default provider/model constants in `src/config.ts` and reuse them in model/controller setup.
2. Thread resolved data paths from `src/cli.ts` into `AssistantController.create()`.
3. Pass explicit memory vector, knowledge vector, knowledge source, and plan paths into their stores.
4. Rework `VectorKnowledgeStore.ingest()` to clear existing chunks and reuse `indexFile()` for all file formats.
5. Preserve `lastModified` metadata in full ingestion and manifest reconstruction so `needsReindex()` is stable across process restarts.
6. Add Vitest configuration to exclude `dist`, and expose `npm test`.
7. Gate `mock_failure` registration behind an explicit test-tool option.

## Verification

- `npm run typecheck`
- `npm test`
- Focused source test run if needed: `./node_modules/.bin/vitest run src/knowledge/__tests__`

## Notes

- Existing uncommitted changes to `src/knowledge/vector-store.ts` are in scope because they fix Obsidian sync behavior when old rows have `lastModified = 0`.
- Planning artifacts for this repair are kept under `.planning` per the project convention.
