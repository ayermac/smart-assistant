# 13-01 Plan: Ink Terminal UI

## Goal

Implement an optional Ink-powered terminal UI that reuses the existing assistant runtime and session persistence.

## Scope

- Add an Ink TUI entry point.
- Share CLI option parsing/session resolution logic where practical.
- Add dependencies and scripts/binary wiring.
- Render streaming assistant text, tool status, and errors.
- Keep the existing readline CLI unchanged for users who prefer the simple mode.

## Implementation Steps

1. Extract reusable CLI runtime helpers from `src/cli.ts` into a shared module:
   - option parsing
   - package version reading
   - session resolution
   - assistant runtime creation
   - optional Obsidian vault watcher setup/teardown
2. Update `src/cli.ts` to use the shared helpers without changing behavior.
3. Add `src/tui.tsx` with an Ink app:
   - header with data dir/session/model hints
   - scrollable message/event log
   - input prompt
   - `/help`, `/exit`, and empty-input behavior
   - prompt-in-progress state and abort on Ctrl+C
4. Add package wiring:
   - dependencies: `ink`, `react`
   - dev dependency for React types if needed
   - `npm run tui`
   - `smart-assistant-tui` binary
   - TypeScript JSX config for React/Ink
5. Add focused tests for helper-level behavior that does not call model providers.
6. Verify:
   - `npm run typecheck`
   - `npm test`
   - `npm run build`

## Review Focus

- TUI must not duplicate persistence logic in a way that diverges from CLI.
- No model/API call should happen during tests.
- Plain CLI must remain stable.
- Dependencies should be minimal and justified.
