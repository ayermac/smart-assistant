# 15-01 Plan: Fix npm Install Dependency Resolution

## Context

Running `npm install` failed with npm ERESOLVE. The first blocker was `@lancedb/lancedb@0.29.0` requiring peer `apache-arrow >=15.0.0 <=18.1.0` while the project declared `apache-arrow ^21.1.0`.

After aligning Arrow, npm also surfaced optional peer conflicts from unused `@langchain/community` dependencies. The current codebase implements PDF and DOCX loading directly through `pdf-parse` and `mammoth`, so the LangChain packages are no longer needed at runtime.

## Goals

- Make plain `npm install` pass without `--force` or `--legacy-peer-deps`.
- Keep LanceDB on the current project version.
- Use an Apache Arrow version inside LanceDB's peer range.
- Remove unused LangChain dependencies that trigger unrelated optional peer conflicts.
- Keep README aligned with the actual document loader implementation.

## Implementation Steps

1. Change `apache-arrow` from `^21.1.0` to `^18.1.0`.
2. Change `dotenv` from `^17.4.2` to `^16.6.1`, which still supports `quiet: true` and satisfies optional peer chains.
3. Remove unused `@langchain/community` and `@langchain/core` dependencies.
4. Refresh `package-lock.json` with normal npm resolution.
5. Update README and README_CN references from LangChain loaders to built-in `pdf-parse`/`mammoth` loaders.
6. Run install, typecheck, tests, build, and TUI help verification.

## Acceptance Criteria

- `npm install` exits successfully.
- `npm ls apache-arrow @lancedb/lancedb dotenv` shows `apache-arrow@18.1.0`, `@lancedb/lancedb@0.29.0`, and `dotenv@16.6.1`.
- `npm run typecheck`, `npm test`, and `npm run build` pass.
- `node dist/tui.js --help` works.
