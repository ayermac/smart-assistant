# 18-01 Plan: Document Changelog and Production Usage

## Context

The README described development commands such as `npm run tui`, but did not explain how to use the built assistant in a production-like local setup. The changelog also did not include the recent TUI, dependency, Obsidian sync, schema repair, and initialization-input fixes.

## Goals

- Add production usage instructions that do not rely on `npm run` scripts.
- Document compiled `dist` entry points.
- Document local command installation through package binaries.
- Update README and README_CN consistently.
- Add Unreleased changelog sections for the current branch work.
- Verify installed production binaries, not only development scripts.

## Implementation Steps

1. Add a Production Usage section to README.
2. Add a corresponding 生产环境使用 section to README_CN.
3. Document `node dist/cli.js`, `node dist/tui.js`, `smart-assistant`, and `smart-assistant-tui`.
4. Include recommended persistent env vars for data directory and Obsidian vault path.
5. Add Unreleased changelog entries for features, fixes, and documentation.
6. Update the standalone CHANGELOG.md, not only README changelog snippets.
7. Verify markdown diff and run lightweight command checks.
8. If installed binary verification exposes a production entrypoint bug, fix it in the narrowest shared entrypoint path.

## Acceptance Criteria

- README and README_CN distinguish development scripts from production/local-installed commands.
- README, README_CN, and CHANGELOG.md include the recent branch changes.
- Installed `smart-assistant --help` and `smart-assistant-tui --help` print usage after local global install.
- `.planning` contains the documentation update record.
