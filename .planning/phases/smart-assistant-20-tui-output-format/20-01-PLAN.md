# 20-01 Plan: Improve TUI Output Formatting

## Context

The Ink TUI displayed assistant responses as raw Markdown text in a single flowing row. Long Chinese/English mixed lines wrapped awkwardly, source citations appeared as raw `According to ...`, and numbered-list continuations were hard to scan.

## Goals

- Keep the TUI terminal-first and compact.
- Use a fixed left prefix column for `system>`, `you>`, `assistant>`, `tool>`, and `error>`.
- Wrap assistant output by visible terminal width.
- Clean lightweight Markdown artifacts from assistant text without changing the model response semantics.
- Display local knowledge citations as `来源：...`.
- Indent continuation lines for numbered and bullet lists.

## Implementation Steps

1. Add a pure TUI formatting module for log normalization and wrapping.
2. Update Ink rendering to consume formatted visual rows instead of raw log entries.
3. Slice visible output by rendered terminal rows, not by event count.
4. Add tests for citation cleanup, list wrapping, and continuation-prefix behavior.
5. Update README, README_CN, CHANGELOG, and planning records.

## Acceptance Criteria

- Long assistant answers wrap within the terminal content column.
- Only the first visual row of a log entry shows the speaker prefix.
- Numbered-list continuation rows are indented under the list text.
- Local knowledge citations are displayed as `来源：path > heading：`.
- Typecheck, tests, build, and CLI/TUI help checks pass.
