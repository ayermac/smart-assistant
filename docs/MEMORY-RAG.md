# Memory 与 RAG

## 区分

- Memory：助手应该长期记住什么
- RAG：助手应该从知识库查什么
- Planning：助手如何拆解和跟踪任务

## Memory 设计

第一版把 memory 分成两层：

- 短期记忆：当前 session 的消息和状态
- 长期记忆：用户偏好、稳定事实、重要任务结论

长期记忆只保存明确值得复用的信息，不保存全部对话。

## RAG 设计

第一版只支持本地 Markdown/text：

- ingest 本地文件
- chunk
- 建索引
- 检索相关片段
- 拼接到上下文

## 第一版不做什么

- 不做 PDF/docx 解析
- 不做网页爬取
- 不做云知识库同步
- 不做复杂图谱检索

## 推荐策略

- 先用简单关键词/分段检索验证流程
- 再决定是否升级 embedding + vector store
- memory 和 RAG 保持接口分离，避免互相污染

## 接口边界

- memory 接口负责写入和回忆长期事实
- RAG 接口负责查询文档片段
- planner 不直接访问存储，只消费 memory / RAG 的结果

## 数据原则

- 只保存必要信息
- 可追溯但不冗余
- 结果要能被测试复现
