# 路线图

## Phase 0：文档和骨架

- 建立产品文档
- 确定 CLI 入口
- 确定技术栈
- 锁定 memory/RAG/planning 的边界

退出标准：
- 文档齐全
- 每个模块的职责明确
- 后续实现不需要再重新定义方向

## Phase 1：MVP agent

- CLI 对话
- `pi-ai` 接模型
- `pi-agent-core` 跑 loop
- 基础工具接入

退出标准：
- 能持续对话
- 能流式输出
- 能调用至少一个工具

## Phase 2：Memory

- 短期会话保存
- 长期记忆写入
- 长期记忆检索

退出标准：
- 能写入一条长期记忆
- 能在后续会话回忆出来

## Phase 3：RAG

- 本地 Markdown/text 索引
- 检索相关片段
- 上下文拼接

退出标准：
- 能从知识库查到相关内容
- 检索不到时能保守回答

## Phase 4：Planning

- 任务拆解
- 计划状态更新
- 失败恢复

退出标准：
- 能输出结构化计划
- 能更新任务状态

## Phase 5：可选扩展

- Web/API 入口
- 更强的检索策略
- 多 agent 协作
- 云端/同步存储

退出标准：
- 扩展能力不影响 CLI 主线稳定性
